## 功能概述
本计划旨在修复和统一个人信息页面的布局与头像交互，确保：
- 个人信息页面以 SubLayout 作为唯一布局模板，移除冗余 Header。
- 头像区域点击仅弹出“暂不支持头像修改”提示，无需预留上传等功能。
- 仅聚焦个人信息页，不涉及其他页面迁移。

---

## Phase 1: 个人信息页面统一采用 SubLayout

### 目标
- PersonalInfo.jsx 以 SubLayout 为根组件，移除冗余 Header 引用，内容区作为 children 传入。
- title 仅为“个人信息”，无副标题、无 rightActions。

### 修改文件清单
|文件路径|修改类型|说明|
|---|---|---|
|src/pages/my/PersonalInfo.jsx|重构|以 SubLayout 包裹全部内容，移除 Header 相关代码|

### 具体变更
- 移除对 Header 的直接引用。
- 以 `<SubLayout title="个人信息">...</SubLayout>` 包裹页面内容。
- 保持内容区结构不变，仅作为 children 传入。

### 成功标准
- 自动验证：页面无编译/运行报错，布局结构与 SubLayout 规范一致。
- 手动验证：页面头部风格与其他次级页面一致，无重复头部。

---

## Phase 2: 头像点击提示功能

### 目标
- 头像区域点击弹出“暂不支持头像修改”提示。
- 无需预留上传、裁剪等 UI 或接口。

### 修改文件清单
|文件路径|修改类型|说明|
|---|---|---|
|src/pages/my/PersonalInfo.jsx|调整|头像 ListMenu.Item 的 onClick 行为|

### 具体变更
- 头像 ListMenu.Item 的 onClick 设置为 `() => message.info('暂不支持头像修改')`。
- 保持头像样式与 ListMenu.Item 统一，无需特殊样式。

### 成功标准
- 自动验证：无编译/运行报错。
- 手动验证：点击头像弹出正确提示，无其他 UI 变化。

---

## Phase 3: SubLayout 及 Header 结构确认

### 目标
- 确认 SubLayout 内部已集成 Header，支持 title、children 等 props。
- 页面无需重复引入 Header。

### 修改文件清单
|文件路径|修改类型|说明|
|---|---|---|
|src/layouts/SubLayout.jsx|查验/微调|如有必要，保证 title、children 正常渲染|

### 具体变更
- 检查 SubLayout 是否已集成 Header 并支持 title、children。
- 如有异常，微调 SubLayout 以保证兼容性。

### 成功标准
- 自动验证：SubLayout 相关页面均无报错。
- 手动验证：个人信息页头部与其他次级页面一致。
