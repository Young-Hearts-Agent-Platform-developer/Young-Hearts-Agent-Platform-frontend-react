# 首页实现与可访问性 研究报告

## 研究日期
2026-01-12

## 研究问题
如何实现“心青年”智能体平台的首页（Home），并确保其可访问性与设计一致性。

---

## 发现摘要
- 首页采用“Logo+1+2”布局，上半为品牌Logo区，下半为主入口（智能体咨询）和两个副入口（查资料/找帮手），底部为固定Tab导航，仅含“首页”“我的”。
- 设计强调移动优先、响应式、无障碍（高对比度、可聚焦、aria标签），主色#4A90E2，背景#F4F6F8，文本#2C3E50。
- 现有 src/pages/home/ 目录为空，src/components/、src/layouts/ 也无现成通用组件，需新建 Logo区、入口卡片、底部导航等组件。
- 首页跳转依赖 React Router，主入口跳转 /consultation，副入口跳转 /knowledge、/community，底部Tab切换 /、/my。
- 组件需支持触控与键盘操作，卡片/按钮需有聚焦、hover、active、disabled等状态，保证无障碍。
- 文案与图标已生成初稿，首页仅需支持中文，组件库采用 Ant Design Mobile。

## 当前实现分析

- 目录结构已预留 src/pages/home/，但无实际实现文件，已新建 Home.jsx/HomeLayout.jsx 及核心组件。
- 通用组件（Logo区、入口卡片、底部导航）已放于 src/components/，页面级布局（含底部栏）已放于 src/layouts/。
- 首页作为一级页面，采用 HomeLayout（含底部导航），内容区为 Logo区+主副入口卡片。
- 路由与跳转依赖 React Router v6，主入口跳转 /consultation，副入口跳转 /knowledge、/community，底部Tab切换 /、/my。
- 设计文档对无障碍、响应式、动效、状态管理有详细要求，已在组件初稿中体现。
- 文案与图标已生成初稿，首页仅需支持中文，Logo 使用 png 格式。
- 组件库采用 Ant Design Mobile，已在实现中引用。

---

### 核心流程（文字描述）

1. 进入首页 `/`，渲染 HomeLayout（含底部导航）。
2. 上半区渲染 Logo区（品牌Logo+名称+副标题）。
3. 下半区渲染主入口卡片（智能体咨询，点击跳转 /consultation）。
4. 主入口下方并排渲染两个副入口卡片（查资料/找帮手，分别跳转 /knowledge、/community）。
5. 底部固定TabBar，含“首页”“我的”，点击切换路由。
6. 所有卡片/按钮支持键盘聚焦、aria-label、hover/active/focus/disabled等状态，保证无障碍。

---

### 关键代码片段（建议实现）

- src/pages/home/Home.jsx：页面主入口，组合 Logo区、主副入口卡片。
- src/components/LogoSection.jsx：Logo区组件。
- src/components/MainEntryCard.jsx：主入口卡片组件。
- src/components/SubEntryCard.jsx：副入口卡片组件。
- src/components/BottomNav.jsx：底部导航栏组件。
- src/layouts/HomeLayout.jsx：一级页面布局，含底部导航。

## 架构洞察

- 推荐先实现通用组件（Logo区、入口卡片、底部导航），后组合为首页页面。
- 路由与布局解耦，便于后续扩展与维护。
- 无障碍与响应式需全流程覆盖，建议开发阶段即用辅助工具检测。
- 文案与图标可持续优化，支持后续产品/设计反馈。

---

## 潜在风险和边缘情况

- 目录虽已预留，但无现成实现，需从零搭建组件与页面（已完成初稿）。
- 路由与布局未初始化，需同步配置 router 与 layouts。
- 无障碍与响应式如未严格实现，易影响目标用户体验。
- 图标与插图素材需与产品/设计确认风格与版权。

---

## 开放问题

- 首页主入口、副入口的具体文案与图标已生成初稿，后续可根据产品反馈优化。
- Logo 已有 png 格式，建议统一放置于 src/assets/ 并在 LogoSection.jsx 中引用。
- 首页仅需支持中文，无需国际化。
- 组件库已确定为 Ant Design Mobile。
