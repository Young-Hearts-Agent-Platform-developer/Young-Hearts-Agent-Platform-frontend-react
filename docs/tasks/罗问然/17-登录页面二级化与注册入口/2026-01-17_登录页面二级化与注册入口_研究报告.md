## 研究问题

将登录页面转化为二级页面，并添加“没有账户？注册新的账户”的入口（本次仅关注登录页面UI调整，不涉及注册页面和注册逻辑实现）。

## 发现摘要

- 当前登录页面为独立页面，未使用二级页面布局（SubLayout）。
- 登录页面未提供“注册新账户”入口。
- 可通过引入 SubLayout 组件包裹登录表单，并在表单下方添加注册入口按钮或链接。

## 当前实现分析

### 登录页面结构
- 组件未使用 SubLayout，直接渲染表单。
- 未包含注册相关 UI 或跳转逻辑。


### 路由结构
- /auth/login 路由已配置。

### SubLayout 结构
- 提供顶部带返回的头部，主内容区为 children。

### 核心流程

```
用户访问 /auth/login
→ 渲染 Login 组件（当前未包裹 SubLayout）
→ 用户输入用户名/密码，提交表单
→ 登录成功后跳转 /my 或来源页
→ 未登录时无注册入口
（建议：用 SubLayout 包裹表单，并在表单下方添加“没有账户？注册新账户”按钮或链接，跳转到注册页或弹窗，具体跳转逻辑本次不实现）
```

## 关键代码片段

- 登录页面未用 SubLayout：src/pages/auth/login.jsx L1-L169
- SubLayout 组件结构：src/layouts/SubLayout.jsx L1-L40

## 架构洞察
- 登录页面建议统一为二级页面风格，提升一致性。
- 注册入口可用按钮或链接形式，放置于登录表单下方。

## 潜在风险和边缘情况
- 仅添加注册入口UI，若后续未补充注册页面，用户体验可能不完整。
- 登录/注册切换需考虑移动端交互友好性。

## 开放问题
- 注册入口跳转目标（页面/弹窗）及后续交互需后续明确。
