## **1. 页面概述**
- **用户目标**：家属或服务者查看个人资料、快速进入历史工单、访问统一服务中心与切换身份/角色。
- **关键操作**：
  - **查看/编辑个人资料**：头像、昵称、身份标签。
  - **访问历史工单**：快速跳转到 `/my/tickets` 列表与单条详情。
  - **进入统一服务中心**：对志愿者/专家/管理员展示入口卡片并可跳转到 `/workspace`。

## **2. 视觉布局**
- **目标前端类型**：Mobile H5（优先），次级提供 PC Web 建议。
- **整体结构**：
  - Header（一级Header，含页面标题与右侧操作）
  - 主内容（Profile Header → 快捷入口卡片 → 历史工单预览 → 其他账号动作）
  - Bottom Tab（底部一级页面导航，常驻：首页 / 个人中心）
- **容器参数（Mobile）**：
  - 视口宽度：375px（设计基准）。
  - 页面主体左右 Padding：16px。
  - 栅格：单列流式布局，卡片左右内边距 16px，卡片之间垂直间距 12px。
  - 底部安全区（iOS 等）：保留 12–20px 空白。
- **容器参数（PC 建议）**：
  - 最大宽度：1200px，水平居中。
  - 页面主体 Padding：24px–32px。

## **3. 组件细节**
（按从上到下、从左到右顺序描述）

### 3.1 Header（一级 Header）
- **组件名**：Header [容器]
- **尺寸与位置**：高度 56px，顶边距 0，左右 Padding 16px。
- **样式细节**：背景色 `#FFFFFF`（或透明），底部边框 `1px solid #E6EDF3`（比背景略深以分割）。
- **内容规范**：标题字号 16px、字重 600、颜色 `#2C3E50`，居中。右侧可放“设置/编辑”图标，图标尺寸 20x20px，色 `#607080`。
- **状态样式**：按下态 Icon 变为 `#4A90E2`（主色），长按无特殊动效。

### 3.2 Profile Header（个人信息卡）
- **组件名**：Profile Header [卡片]
- **尺寸与位置**：高度 160px；上下 Margin：12px；左右 Padding：16px；圆角 12px。
- **背景与边框**：背景 `#FFFFFF`，阴影 `0 4px 10px rgba(46,60,75,0.06)`，边框无或 `1px solid rgba(44,62,80,0.04)`。
- **内部布局**：左侧头像，右侧垂直信息区域。
  - **头像**：80x80px，圆形（border-radius: 50%）；外侧留白 12px。默认占位图为浅灰圆形（背景 `#F4F6F8`），占位图标色 `#9CA3AF`。
  - **昵称**：字号 18px、字重 600、颜色 `#2C3E50`，顶部对齐头像中心偏上，间距 8px。
  - **身份标签**：小标签高度 22px，内边距 8px；背景 `#F0F7FF`，文本色 `#4A90E2`，圆角 8px，字号 12px。标签示例：志愿者 / 家属 / 专家。
  - **编辑入口**：位于右上角浮动的“编辑”小按钮，尺寸 36x36px，背景 `#FFFFFF`、边框 `1px solid #E6EDF3`、图标色 `#4A90E2`。
- **交互**：点击头像或“编辑”进入资料编辑页，头像悬停/触摸时出现 48% 黑色遮罩及“更换”图标（移动端长按唤起上传选择）。

### 3.3 快捷功能区（1 卡 + 2 小卡布局）
- **组件名**：Quick Entrance Grid [栅格]
- **布局**：主卡（大卡）占满一行高度 100px（视觉主入口）；下方并排两个小卡高度 84px，各占 50% 宽度，中间间距 12px。
- **主卡（智能体咨询）**：
  - 尺寸：宽度 = 容器宽 - 32px；高度 100px；圆角 10px。
  - 背景：线性渐变或浅色块，主要色 `#4A90E2` 背景渐变至 `#3A7BD5`，文字白色 `#FFFFFF`。
  - 内容：左侧大图标（48x48），中间标题 18px（字重700），下方副文本 12px（如“点击开始对话”）。
  - 交互：按下态整体透明度降 8%；短时 Loading 态显示旋转指示器与文案“连接中...”。
- **小卡（查资料 / 找帮手）**：
  - 背景 `#FFFFFF`，边框 `1px solid #E6EDF3`，圆角 10px；图标 28x28，标题 14px，描述 12px。
  - 悬停/按下：图标背景变为 `#F4F8FF`，标题文字色 `#4A90E2`。

### 3.4 历史工单预览（My Tickets Preview）
- **组件名**：Tickets Preview [列表卡片]
- **尺寸与位置**：卡片高度 112px；内部上下 Padding 12px；卡片间距 12px。
- **样式细节**：背景 `#FFFFFF`，左侧状态色条 6px 宽（绿色 `#50E3C2` / 橙 `#F5A623` / 灰 `#CBD5E1`），标题 15px、摘要 13px、时间 12px（颜色 `#9CA3AF`）。
- **内容规范**：最多展示 3 条预览，底部右侧有“查看全部”链接（主色 `#4A90E2`）。
- **交互**：点击条目进入工单详情 `/my/tickets/:id`；列表为空显示 Empty State（见 4.2）。

### 3.5 统一服务中心入口（Conditional Card）
- **组件名**：Service Center Entry [卡片]
- **可见性**：仅当用户角色为 `Support`（志愿者/专家/管理员）时展示。
- **尺寸与位置**：高度 92px，左右 Padding 16px，圆角 10px，垂直间距 12px。
- **样式细节**：背景 `#FFFFFF`，左侧图标区（56x56）使用辅助色 `#50E3C2` 渐变；右侧为主文案（16px）与子文案（12px）。
- **交互**：点击卡片进入 `/workspace`；按下态背景微暗（opacity 0.96）；长按无额外交互。

### 3.6 账号动作与安全（底部）
- **组件名**：Account Actions [列表]
- **项示例**：切换角色 / 安全设置 / 退出登录。
- **尺寸与样式**：每项高度 52px，文本 15px，右侧箭头图标 16x16，背景 `#FFFFFF`，分割线 `1px solid #F1F5F8`。
- **删除操作（退出登录）**：文本颜色 `#F43F5E`（Danger），按下弹出二次确认模态。

## **4. 交互说明**

### 4.1 全局导航与返回逻辑
- 所有次级页面左上角 `<` 执行 `history.back()`；若无历史栈兜底跳转至首页 `/`。
- 底部 Tab 常驻，点击对应 tab 切换路由，不产生堆栈回退（替换历史）。

### 4.2 表单/列表状态与提示
- **Loading 态**：按钮变为不可点击并显示微型菊花，文案如“保存中...”或“连接中...”。
- **Empty 态（历史工单为空）**：展示 1 条高度 160px 的占位卡片，图标居中，文案“暂无历史工单”，下方动作按钮“发起工单”（主色）。
- **错误态**：短文错误提示位于控件下方，字体 12px，颜色 `#F43F5E`。长期错误在顶部弹出 Toast（背景 `#FEE2E2`，文字 `#7F1D1D`）。

### 4.3 微动效
- **按钮按下**：缩放到 0.98，过渡 120ms。
- **卡片出现**：Y 轴轻微位移 8px → 0，透明度 0 → 1，过渡 220ms。
- **头像更换**：上传成功后头像用 250ms 的淡入淡出过渡。

## **5. 适配与响应式**
- **Mobile H5（主）**：单列流式；触控目标最小 44x44px；侧边栏不存在；Bottom Tab 固定在视口底部。键盘弹出时使用 Viewport Resize 保持输入框可见。
- **PC Web（建议）**：将主体容器最大宽度设为 1200px，Profile Header 宽度扩展为 100% 横向布局（头像左、信息右）；快捷入口改为三列 Grid（1 主 + 2 支持改为 1 行 3 列可选）；悬停态补充 Tooltip 和更多交互提示。
- **Accessibility（无障碍）**：文本对比度满足 WCAG AA：主文本色 `#2C3E50` 与背景 `#F4F6F8` 的组合通过可读性检查；所有关键图标带 aria-label，交互控件可通过键盘聚焦。
