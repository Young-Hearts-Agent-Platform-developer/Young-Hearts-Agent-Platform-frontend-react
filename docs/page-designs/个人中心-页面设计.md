## **1. 页面概述**
- **用户目标**：家属或服务者查看个人资料、快速进入历史工单、访问统一服务中心与切换身份/角色。
- **关键操作**：
  - **查看/编辑个人资料**：头像、昵称、身份标签。
  - **访问历史工单**：快速跳转到 `/my/tickets` 列表与单条详情。
  - **进入统一服务中心**：对志愿者/专家/管理员展示入口卡片并可跳转到 `/workspace`。

## 2. 视觉布局
- **目标前端类型**：Mobile H5（主），兼容 PC Web。
- **整体结构**：
  - 顶部为 Header 区域，宽度占满页面上方，内容左右留有适度留白。
  - 主内容区自上而下依次为：个人信息卡、快捷入口区、历史工单预览、（条件渲染）统一服务中心入口、账号动作区。
  - 主内容区占据页面大部分空间，底部为常驻导航栏。
  - 各区块之间有明显分隔，卡片之间垂直间距适中，整体布局宽松，视觉重心突出个人信息与主入口。
  - PC Web 建议：主内容区居中显示，宽度相对更宽，快捷入口区可拓展为多列栅格。

## **3. 组件细节**
（按从上到下、从左到右顺序描述）

### 3.1 Header [容器]
- **相对尺寸与位置**：顶部固定，宽度占满页面上方，内容左右有适度留白。
- **样式细节**：背景为白色或透明，底部分隔线弱化，圆角无，阴影极弱。
- **内容规范**：标题居中，字号和字重在页面中较为突出，主文本色。右侧为设置/编辑图标，悬停/按下主色高亮。

### 3.2 Profile Header [卡片]
- **相对尺寸与位置**：主内容区视觉重心，卡片宽度自适应，左右居中。
- **样式细节**：背景为白色，圆角柔和，阴影轻微，边框极浅。头像大且圆形，昵称与身份标签分层次排布。
- **内容规范**：
  - 头像：圆形，位于卡片左侧，外侧留白充足，默认灰色占位。
  - 昵称：主文本色，字号和字重在卡片中最突出，紧邻头像右侧。
  - 身份标签：主色背景，圆角，文本色主色，紧随昵称下方。
  - 编辑入口：右上角悬浮，主色描边，图标点缀。
- **状态样式**：头像悬停/长按出现半透明遮罩与更换提示，编辑按钮按下主色高亮。

### 3.3 快捷功能区（Quick Entrance Grid）
- **相对尺寸与位置**：主卡占据一行，两个小卡并排，各占一半宽度。
- **样式细节**：主卡主色渐变，圆角柔和，图标大且醒目，副文本弱化。小卡白底主色描边，图标点缀，悬停主色高亮。
- **内容规范**：主卡为“智能体咨询”，小卡为“查资料”“找帮手”，均有图标与简短描述。
- **状态样式**：主卡按下透明度降低，Loading 态旋转指示器。小卡悬停主色描边，标题高亮。

### 3.4 历史工单预览（Tickets Preview）
- **相对尺寸与位置**：主内容区下方，卡片纵向排列，单列流式。
- **样式细节**：白底圆角，左侧状态色条区分工单状态，内容分主副文本，时间信息弱化。
- **内容规范**：最多展示 3 条，底部有“查看全部”主色链接。
- **状态样式**：卡片悬停阴影增强，点击进入详情。空列表时展示美观 Empty State。

### 3.5 统一服务中心入口（Service Center Entry）
- **相对尺寸与位置**：历史工单预览下方，单独一行，仅 Support 角色可见。
- **样式细节**：白底，左侧大图标区辅助色渐变，右侧主副文案分层，圆角柔和。
- **内容规范**：主文案突出“进入统一服务中心”，副文案简要说明。
- **状态样式**：卡片按下微暗，点击跳转。

### 3.6 Account Actions [列表]
- **相对尺寸与位置**：页面底部，紧邻底部导航上方，宽度与主内容区一致。
- **样式细节**：白底分割线，主文本色，右侧箭头主色描边。
- **内容规范**：切换角色、安全设置、退出登录等。退出登录为危险色，点击弹出确认。

## **4. 交互说明**

### 4.1 全局导航与返回逻辑
- 所有次级页面左上角 `<` 执行 `history.back()`；若无历史栈兜底跳转至首页 `/`。
- 底部 Tab 常驻，点击对应 tab 切换路由，不产生堆栈回退（替换历史）。

### 4.2 表单/列表状态与提示
- **Loading 态**：按钮变为不可点击并显示微型菊花，文案如“保存中...”或“连接中...”。
- **Empty 态（历史工单为空）**：展示 1 条高度 160px 的占位卡片，图标居中，文案“暂无历史工单”，下方动作按钮“发起工单”（主色）。
- **错误态**：短文错误提示位于控件下方，字体 12px，颜色 `#F43F5E`。长期错误在顶部弹出 Toast（背景 `#FEE2E2`，文字 `#7F1D1D`）。

### 4.3 微动效
- **按钮按下**：缩放到 0.98，过渡 120ms。
- **卡片出现**：Y 轴轻微位移 8px → 0，透明度 0 → 1，过渡 220ms。
- **头像更换**：上传成功后头像用 250ms 的淡入淡出过渡。

## **5. 适配与响应式**
- **Mobile H5（主）**：单列流式；触控目标最小 44x44px；侧边栏不存在；Bottom Tab 固定在视口底部。键盘弹出时使用 Viewport Resize 保持输入框可见。
- **PC Web（建议）**：将主体容器最大宽度设为 1200px，Profile Header 宽度扩展为 100% 横向布局（头像左、信息右）；快捷入口改为三列 Grid（1 主 + 2 支持改为 1 行 3 列可选）；悬停态补充 Tooltip 和更多交互提示。
- **Accessibility（无障碍）**：文本对比度满足 WCAG AA：主文本色 `#2C3E50` 与背景 `#F4F6F8` 的组合通过可读性检查；所有关键图标带 aria-label，交互控件可通过键盘聚焦。
