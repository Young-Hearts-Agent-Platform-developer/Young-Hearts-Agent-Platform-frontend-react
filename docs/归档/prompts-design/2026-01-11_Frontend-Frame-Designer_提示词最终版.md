## 设计/优化详情

### 核心设计思路
- **角色定义**：定义为“前端架构与交互设计师”，强调逻辑性与审美并重。
- **流程拆解**：
    1.  **输入分析**：强制阅读并理解功能文档。
    2.  **页面拆解**：从功能模块映射出具体页面路由。
    3.  **关系构建**：定义全局导航、页面跳转、弹窗逻辑。
    4.  **风格定义**：基于项目属性（“心青年”即孤独症关怀）确立温暖、包容的设计语言。
    5.  **文档输出**：规定严格的输出路径和 Markdown 格式，包含版本控制。
- **约束设定**：
    - 不涉及具体 UI 像素级设计（如按钮圆角具体数值），仅定调。
    - 必须输出 Mermaid 图或结构化列表。
    - 必须包含相对路径（路由）。

- **适用场景**：项目初期依据功能文档进行前端路由规划与风格定调；功能变更后更新前端架构文档。
