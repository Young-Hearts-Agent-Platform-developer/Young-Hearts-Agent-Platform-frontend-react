# “心青年”智能体平台 - H5 前端界面设计方案

## 1. 设计概览

### 1.1 设计理念
基于“心青年”项目的特殊服务对象（孤独症家庭、志愿者、专家），设计遵循以下核心原则：
- **情感化治愈 (Healing)**：界面应传递希望与平静，避免使用产生焦虑的高饱和度色彩。
- **低认知负担 (Cognitive Ease)**：为处于高压力状态下的家属提供极简的操作路径，核心功能“一键直达”。
- **无障碍友好 (Accessibility)**：考虑到用户可能存在的视觉或操作障碍，采用大触控区、高对比度文字与语音交互优先策略。

### 1.2 视觉规范
- **风格关键词**：温暖、信赖、极简、包容。
- **色彩体系**：
    - **主色 (Primary)**：`#4A90E2` (静谧蓝) - 用于主按钮、激活状态，传递专业与冷静。
    - **辅色 (Secondary)**：`#F5A623` (暖阳橙) - 用于重要提示、志愿者服务相关，传递活力与温情。
    - **背景色**：`#F7F8FA` (云朵灰) - 用于页面底色，减少视觉疲劳。
    - **功能色**：`#FF5B5B` (紧急/错误)、`#52C41A` (成功/在线)。
- **排版 (Typography)**：
    - 字体：系统默认无衬线字体 (San Francisco / PingFang SC / Roboto)。
    - 字号：正文不小于 16px，辅助文字不小于 14px，标题强调层级感。
- **图标风格**：圆润线面结合图标，亲和力强，避免尖锐棱角。

### 1.3 交互范式 (H5 Mobile)
- **导航模式**：底部固定标签栏 (Bottom Tab Bar) 承载核心入口。
- **操作习惯**：单手操作优化，关键按钮置于屏幕下半部分（拇指热区）。
- **反馈机制**：操作即时反馈（Toast/Loading/Haptic），特别是AI生成过程中的状态提示。

## 2. 页面清单 (Sitemap)

### A. 患者家属视图 (Client View)
1.  **智能咨询 (Home/Chat)** - 核心入口，RAG 问答界面。
2.  **知识库 (Wiki)** - 知识分类、搜索与文章详情。
3.  **找帮手 (Helpers)** - 专家/志愿者信息墙。
4.  **个人中心 (Profile)** - 历史工单、账号设置、身份切换。

### B. 工作台视图 (Service View - 志愿者/专家/管理员)
*（通过个人中心进入）*
1.  **统一服务中心 (Dashboard)** - 功能入口聚合页。
2.  **任务看板 (Task Board)** - 待办工单、线下班次管理。
3.  **知识贡献 (Contribution)** - 录入/审核表单。

## 3. 页面详述

### 3.1 核心页面：智能咨询 (Page_Home_Chat)
**ID**: `P_Chat_001`
**功能映射**: RAG 智能咨询引擎、语音转文字 input。

- **布局 (Layout)**：
    - **Header**：简洁标题栏，右侧“清空/新对话”图标。
    - **Body**：消息流滚动区域。
    - **Footer**：固定在底部的输入操作区。

- **视觉细节**：
    - **AI 气泡**：左侧显示，背景色 `#FFFFFF`，带有品牌 Logo 头像。回答内容结构化展示（Markdown 渲染），关键信息（如“立即干预”）加粗。
    - **用户气泡**：右侧显示，背景色 `#E6F7FF` (极浅蓝)。
    - **引用来源 (Source)**：AI 回答底部以卡片或折叠标签形式展示“📚 来源：xxx指南”，点击需弹出浮层显示详情。
    - **兜底提示**：若 AI 无法回答，直接在气泡下显示“转人工咨询”按钮（实心橙色按钮）。

- **交互逻辑**：
    - **语音输入**：输入框右侧设置显著的“麦克风”图标，长按录音，松开转文字。考虑到隐私，无需实时发送语音，而是转为文字供用户确认后发送。
    - **思考状态**：发送后，界面展示“正在查阅知识库...”的动态 Loading，给予用户系统正在工作的心理预期。

### 3.2 页面：知识库可视化 (Page_Wiki_List)
**ID**: `P_Wiki_001`
**功能映射**: 知识库查阅、全文检索。

- **布局**：
    - **Top**：大尺寸搜索框（吸顶），支持模糊搜索。
    - **Nav**：横向滚动的胶囊标签栏（分类：#情绪干预 #生活自理 #就业指导）。
    - **Body**：瀑布流或列表展示文章卡片。

- **视觉细节**：
    - **文章卡片**：包含封面图（可选）、标题（最多两行）、摘要（截断）、底部的标签与阅读量。
    - **搜索结果**：高亮匹配关键词。

- **交互**：
    - 点击卡片跳转至**文章详情页 (`P_Wiki_Detail`)**。
    - 详情页右上角需有“分享”按钮。

### 3.3 页面：统一服务中心 (Page_Workspace_Dash)
**ID**: `S_Dash_001`
**功能映射**: 志愿者管理、专家协同、知识贡献入口。
**权限**：仅志愿者、专家、管理员可见。

- **布局**：
    - **Header**：显示用户角色徽章（如“认证专家”）、当前状态切换开关（在线/忙碌/离线）。
    - **Body**：栅格化布局的功能卡片入口 (Grid Menu)。

- **功能卡片 (根据角色动态展示)**：
    - **我的任务**：显示当前进行中的任务数量徽标 (Badge)。
    - **任务大厅**：进入抢单/报名页面。
    - **知识录入**：跳转表单页。
    - **内容审核**：(专家/管理) 待审核条目数。
    - **人员排班**：(管理) 线下活动排班。

- **视觉细节**：
    - 每个卡片采用大图标 + 文字说明，卡片带有轻微投影，易于点击。
    - 状态切换开关使用红/绿指示灯样式，直观反馈服务状态。

### 3.4 页面：任务看板与执行 (Page_Task_Board)
**ID**: `S_Task_001`
**功能映射**: 志愿者/专家接单、处理工单。

- **布局**：
    - **Tabs**：顶部切换 [待处理] [进行中] [已完结]。
    - **List**：工单卡片列表。

- **工单卡片信息**：
    - 标题（家属描述摘要）。
    - 标签（紧急程度、分类）。
    - 时间（距离发布已过去多久，超时标红）。
    - 操作栏：[认领] 或 [查看详情]。

- **交互**：
    - 点击 [认领] 弹出确认框（如果是专家，提示“直接接管”）。
    - 进入 [详情页] 后，底部变为回复框，支持发送图文建议。

### 3.5 页面：找帮手/人员墙 (Page_Public_Wall)
**ID**: `P_Public_001`
**功能映射**: 专家/志愿者信息公开机制。

- **布局**：
    - **Filter**：顶部筛选栏（筛选专家/志愿者、擅长领域）。
    - **List**：人员名片列表。

- **名片设计**：
    - 头像 + 昵称 + 认证标识（V标）。
    - **信息脱敏**：绝**不**显示手机号。若用户设置公开邮箱，显示“✉️ 发送邮件”按钮；否则仅显示擅长领域和服务时长。
    - **状态点**：头像旁显示绿点（在线）或灰点（离线）。

## 4. 无障碍与适配说明 (Accessibility & Adaptation)

### 4.1 字体与可读性
- 支持系统级字体大小缩放 (Rem 布局)。
- 文本颜色与背景对比度至少达到 WCAG AA 标准（4.5:1），确保视力不佳者也能看清。

### 4.2 操作辅助
- **大点击区**：手机端按钮高度不低于 44px (iOS 标准)，避免误触。
- **语音辅助**：聊天页面核心功能必须支持 Screen Reader (读屏软件) 焦点访问，Label 标签清晰（例如“语音输入按钮”，而不是单纯的图标）。

### 4.3 情感化设计细节
- **Loading 动画**：使用柔和的呼吸灯效果，而不是由于系统卡顿产生的生硬转圈。
- **空状态 (Empty State)**：当没有搜索结果或任务时，使用插画（如“暂无内容，休息一下吧”）配合引导文案，减少挫败感。
